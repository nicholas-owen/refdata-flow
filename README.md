This project provides a reproducible, automated workflow for downloading bioinformatics reference datasets (genomes, transcriptomes, and annotations) for multiple species. 

It uses a **two-step architecture** designed for HPC environments:
1. An interactive Python wizard (`resolve.py`) to parse generic queries (like `hg38`) into exact Nextflow requests.
2. A **Nextflow** pipeline that safely runs the parallel downloads on background compute nodes via the **genomepy** engine.

## Prerequisites

- Python 3 with `pip` available (the wrapper script will self-install everything into a hidden virtual environment).
- Nextflow (>= 22.0)
- Conda (or Docker/Singularity) for the Nextflow execution.

## Directory Structure

```
├── bin/
│   ├── download.py          # Core Python script used by Nextflow
│   ├── resolve.py           # Interactive Python pre-flight wizard
│   └── update_refgenie.py   # Post-processing script to automate Refgenie catalog
├── data/
│   └── references/          # Default output directory for genomes
├── main.nf                  # The Nextflow pipeline
├── nextflow.config          # Nextflow configuration profiles
├── README.md
├── requests_raw.csv         # The configuration file you edit
├── requests_resolved.csv    # Generated by resolve.py, used by Nextflow
└── run_pipeline.sh          # Unified wrapper script to run everything
```

## How to use

### Step 1: Write Generic Requests
Edit the `requests_raw.csv` file to list the generic species you want. 
Example `requests_raw.csv`:
```csv
species,query,provider,annotation,aliases
Homo_sapiens,grch38,ensembl,true,hg38;human
Mus_musculus,mouse,,true,mm39;mm
```
*(Note: If you leave `provider` empty like for Mouse, it will search all providers! You can also supply multiple custom aliases separated by semicolons in the `aliases` column.)*

### Step 2: Run the Pipeline Wrapper
Instead of memorizing the Python and Nextflow commands, simply run the provided shell script:

```bash
bash run_pipeline.sh requests_raw.csv --outdir /path/to/your/custom/storage
```
*(If you omit `--outdir`, it defaults to `data/references`)*

This script will automatically:
1. **Resolve**: Parse your `requests_raw.csv`. It connects to the databases and interactively prompts you if there are multiple matches for your generic queries.
2. **Download**: Automatically launch the Nextflow pipeline on the resolved targets using Conda. It natively uses the `-resume` flag so it never redownloads datasets you already completed.
3. **Refgenie Catalog**: Finally, it automatically initializes a `refgenie_export.yaml` for you (if one doesn't exist yet), computes the sequence hashes, securely links your aliases, and cleans up the temporary Nextflow downloads to save space!

### 💡 Pro-Tip: Skipping the Wizard & Caching
The interactive wizard remembers your choices! If you run the pipeline again to add a new species, it will look at `requests_resolved.csv`. If an older species has already been resolved, **it will automatically skip the slow network search and interactive prompt** for that species to save time. 

If you ever *want* to change the assembly for a species, simply delete that specific row from `requests_resolved.csv` (or delete the file entirely) and the wizard will prompt you again!

### 🧹 Pro-Tip: Storage Optimization
Nextflow can generate a lot of background cache in the `.nextflow/` and `work/` directories over time. To instantly purge all logs and pipeline cache to recover your disk space, simply run:
```bash
bash run_pipeline.sh --clean
```

### 🔍 Pro-Tip: Dry-Run Mode
If you want to parse your `requests_raw.csv` and securely resolve exactly which datasets and providers will be used without actually downloading gigabytes of data, run the pipeline in dry-run mode:
```bash
bash run_pipeline.sh --dry-run
```

## Output & Provenance

For each requested assembly, Nextflow will generate an output directory structured securely based on the provider, species, and assembly.

Example:
`/path/to/your/custom/storage/Ensembl/Homo_sapiens/GRCh38.p14/`

Inside each assembly directory, you will find:
- The downloaded compressed `.fa.gz` (and `.gtf.gz` if requested).
- A `README.txt` generated natively by genomepy detailing the source URLs.
- A custom `provenance.json` recording exactly when the dataset was downloaded and what arguments were used, providing an audit trail.

## Refgenie Integration & Provenance

When using `--outdir /reference_data/`, Refgenie will gracefully organize the genomes into a heavily structured vault, using globally agreed sequence hashes to guarantee absolute reproducibility.

### Example Directory Structure

```text
/reference_data/
├── genome_config.yaml              <-- The master database index
│
├── data/                           <-- The Vault (Files stored by cryptographic hash)
│   ├── 2230c53b... (Human Hash)/
│   │   ├── fasta/
│   │   │   └── default/
│   │   │       ├── GRCh38.p14.fa.gz
│   │   │       └── GRCh38.p14.fa.gz.fai
│   │   └── annotation/
│   │       └── default/
│   │           └── GRCh38.p14.annotation.gtf.gz
│   │
│   └── b88a9121... (Mouse Hash)/
│       ├── fasta/
│       │   └── default/
│       │       ├── GRCm39.fa.gz
│       │       └── GRCm39.fa.gz.fai
│       └── annotation/
│           └── default/
│               └── GRCm39.annotation.gtf.gz
│
└── alias/                          <-- Human-readable shortcuts
    ├── GRCh38.p14/                 <-- Official Assembly Alias
    │   ├── fasta -> ../../data/2230c53b.../fasta/default
    │   └── annotation -> ../../data/2230c53b.../annotation/default
    │
    ├── grch38/                     <-- Your custom CSV Query Alias!
    │   ├── fasta -> ../GRCh38.p14/fasta
    │   └── annotation -> ../GRCh38.p14/annotation
    │
    ├── hg38/                       <-- Semicolon-separated CSV Custom Aliases!
    │   ├── fasta -> ../GRCh38.p14/fasta
    │   └── annotation -> ../GRCh38.p14/annotation
```

### Benefits for HPC Research
1. **Zero Duplication:** If two researchers run an analysis—one explicitly asking for `GRCh38.p14` and the other lazily asking for `grch38`—they are both funneled through symlinks to the *exact same physical file* in the `data/` vault.
2. **Absolute Provenance:** Researchers can query Refgenie to get the sequence hash (e.g., `2230c53b...`), which serves as a mathematically indisputable fingerprint of the exact DNA sequences they used for their publication.
